# ショップの追加・更新ワークフロー

このワークフローは、海外通販サイト（ショップ）の新規追加、または既存情報の更新を行う手順です。USERから「ショップを追加して」「〇〇を更新して」と依頼された際は、この手順に沿って進行してください。

## Step 1: ショップ情報の整理とCSV照合
USERからショップURLや名前など一部の基本情報が提供された場合、AIエージェントは自律的に不足情報を調査するか、USERに以下をヒアリングします。
また、必ず `docs/assets/skimlinks_merchants.csv` を開き、対象サイトのドメインが含まれているか確認してください。

- ショップ名
- 公式サイトURL（ドメインをCSVで検索！）
- 発送元国（例: イギリス、アメリカ）
- カテゴリ（例: セレクト・トレンド、ラグジュアリー・百貨店 等）
- 概要メモ（おおまかな売りのポイント）
- **is_affiliate** (CSVに含まれていれば true、なければ false)

## Step 2: 公式情報の調査
以下の3点について、実際の公式サイト情報などをもとにテキストで解説を作成します。
※ 情報が不確かな場合や、到着時に関税などを支払う必要がある場合（DDU）は、ユーザーの混乱を避けるため解説文の末尾等に「詳細は必ず公式サイトをご確認ください。」といった注意書きを添えてください。
1. **日本へ直送可能か（shipping_guide）**
2. **関税は込みか（tax_guide）**
3. **送料の目安（fee_guide）**

## Step 3: SQLの生成と確認
Step1とStep2で収集した情報を元に、以下のフォーマットでデータを整理します。この際、USERにプレビューを提示して、内容に問題がないか確認を取ります。

```sql
-- 新規追加の場合の例
INSERT INTO public.shops (slug, name, url, country, category, image_url, description, shipping_guide, tax_guide, fee_guide, is_affiliate)
VALUES (
  'ショップのslug（例: asos）',
  'ショップ名',
  '公式サイトのトップURL',
  '発送元国',
  'カテゴリ',
  '画像URL（あれば）',
  'ここに概要を入れる',
  'ここに日本発送についてのテキストを入れる',
  'ここに関税システムについてのテキストを入れる',
  'ここに配送料金・日数についてのテキストを入れる',
  true -- CSVに含まれていればtrue
);
```

## Step 4: データベース（Supabase）への反映
USERから内容の「OK」が出た場合、Node.jsのスクリプトを作成・実行し、Supabase（`shops` テーブル）へ追加または更新を行います。
実行後、ターミナルで成功ログが確認できたら、完了報告を行います。

## Step 5: サムネイルの設定
ショップ一覧で見栄えを良くするため、[ショップサムネイル管理ワークフロー](manage_shop_thumbnails.md) を参照して、公式サイトのスクリーンショットを登録してください。
