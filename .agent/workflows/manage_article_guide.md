# 解説記事・ガイド管理ワークフロー

このワークフローは、海外通販の基礎知識などの解説記事を追加し、`/guide` ページへ反映させる手順です。

## Step 1: 記事の作成 (Supabase)
Supabaseの `posts` テーブルに新規記事を追加します。
- `slug`: URL末尾になる一意の文字列
- `title`: 記事タイトル
- `content`: HTML形式の本文
- `category`: 'ガイド' （基礎知識系の場合）
- `thumbnail_url`: 基本的に無し（箇条書き表示用）

## Step 2: ガイドページへのリンク追加
新しい記事を `/guide` ページに表示させるには、物理ファイルを手動で更新する必要があります。

1. `app/guide/page.tsx` を開きます。
2. `const sections = [...]` 配列の中から、適切なセクション（基礎知識、はじめての海外通販、トラブル対応など）を見つけます。
3. そのセクションの `articles` 配列に、追加した記事の `{ title, slug }` を追記します。

```tsx
// 例
{ title: '新しい記事のタイトル', slug: 'added-article-slug' },
```

## Step 3: 確認・反映
1. ローカルで `npm run dev` を起動し、`/guide` ページに正しくリンクが表示されているか確認します。
2. ホバー時の演出などが正しく動作するか確認します。
3. 修正を GitHub にプッシュして本番環境へ反映させます。

## 注意事項
- `app/guide/page.tsx` は `'use client'` 指定されているため、メタデータの直接エクスポートはできません。
- 記事の並び順は `page.tsx` 内の配列の定義順に従います。
