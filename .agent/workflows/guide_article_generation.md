# ガイド・特集記事の自動生成フロー (Ver 3.2 - Revenue & Quality Standard)

このワークフローは、専門性と収益性を両立させたメガガイド記事を生成・投稿するための手順書です。

## Step 1: 構成案の作成とショップ選定（収益性優先）
1. **テーマ決定**: ターゲットキーワード（例：`海外通販 自転車 おすすめ`）を決定。
2. **構成案（Brief）の作成**: `tmp/article-factory/[slug]-brief.json` を作成。
3. **ショップ選定ルール（IMPORTANT）**:
   - ユーザーの要望に合致するショップの中から、**アフィリエイト報酬（収益性）が発生するショップを最優先で上位に配置**する。
   - 収益性がないが認知度が高いショップ（直販のみ等）は、情報の補完として記事の後半に配置する。
4. **リンクの生存確認（MANDATORY）**: 指定した全ショップの公式サイトURLを実際に開き、**403エラーが出ていないか**を確認する。

---

## Step 2: 執筆素材（Draft）の生成と画像の準備
1. **Draft作成**: `npm run generate-article-draft` を実行。
2. **アイキャッチ画像の生成（MANDATORY）**: `generate_image` を使用して、記事固有の高品質ビジュアルを作成。
   - ルール: ショップのロゴやUI画像の流用禁止。16:9のWebP。

---

## Step 3: AI執筆と品質・収益性ルール
`tmp/article-factory/drafts/[slug].json` の `html` を編集。

### 執筆ルール（MANDATORY）:
- **文字数**: 2200文字以上（メガガイドは3500文字推奨）。
- **外部リンク（商用リンク）**:
  - 公式サイト等へのリンクは必ず **`target='_blank' rel='noopener'`** を設定する。
  - アフィリエイトリンクが利用可能な場合は、ボタン（CTA）に優先的に設定する。
- **内部リンク（回遊性向上）**:
  - ショップ個別詳細ページへのリンクは **「ショップ詳細：[ショップ名]のガイド」** という固定文言で `<a>` タグを記述する。
- **ショップ紹介の黄金順序**:
  1. ショップ名（`<h3>`）
  2. 生成画像（`<img>`）
  3. 具体的な解説文（特に関税、配送の専門的な体験談）
  4. 公式サイトへのCTAボタン（アフィリエイト考慮）
  5. ショップ個別ガイドへの内部リンク

---

## Step 4: 投稿と画像自動マイグレーション
```bash
npm run publish-generated-article -- --brief [BRIEF_PATH] --article-file [DRAFT_PATH]
```

## Step 5: インデックス申請
```bash
npm run submit-indexing -- https://original-price.com/articles/[slug]
```
