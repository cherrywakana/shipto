# ガイド・特集記事の自動生成フロー (Ver 4.3 - HTML Table Enforcement)

このワークフローは、検索結果で1位を獲得することを目的とし、技術的・機能的に完璧なコンテンツを生成するための手順書です。

## Step 1: ショップ選定・収集とSEO戦略
## Step 1: ショップ選定・収集とSEO戦略
1. **ショップ選定の自動化（中間テーブルの活用）**: 記事にするブランドが決まったら、**必ず事前に Supabase の `shop_brands` テーブルをクエリ照会すること**。既に自動クローラー（GitHub Actions）が `status: 'found'` として収集済みの店舗リストと `brand_url` が存在するため、それらの実データをロードして記事に使う対象ショップを決定すること。エージェント自身の推測によるハードコーディングは禁止する。
2. **アフィリエイト最優先の配置（厳格なルール）**: `shop_brands` から抽出したショップ群の中で、「Skimlinks等でアフィリエイト連携があるか」「日本への直接配送が可能でCVRが高いか」を基準としてランク付けを行い、最も収益性の高いショップを必ずTop 3〜Top 5に優先配置する。
3. **既存DBデータの再利用（厳密な確認）**: `shops` テーブルから各種データ（特に `image_url`）を100%継承・再利用し、極力プレースホルダーの生成を回避すること。
4. **構成**: 比較表、カテゴリー別深掘り（散文）、実務ガイド（関税・サイズ）、FAQを必須とする。

## Step 2: ビジュアルとデータの配備
1. **画像**: ショップごとに **実機スクリーンショット** を必ず使用（AI生成厳禁）。
2. **比較表の記述（最重要）**: 
   - **Markdown の表でレイアウトが崩れる場合、またはPCでの表示を完全に安定させる必要がある場合は、必ず HTML の `<table>` タグを使用して記述すること。**
   - テーブルには必ず `style="width: 100%; border-collapse: collapse;"` を付与し、全幅表示を強制する。
   - ヘッダーには適切な背景色（例：`background: #000; color: #fff;`）をインラインスタイルで適用し、デザインの統一を図る。

## Step 3: プロフェッショナルな執筆
1. **文体**: プロの編集者による「散文」で解説し、箇み書きを多用しない。
2. **強調**: 強調には **必ず `<strong>...</strong>` タグを使用する。**
3. **CTAボタンの100%設置**:
   - 比較表にあるすべてのショップに対し、本文中に詳細セクションを設ける。
   - すべての個別セクションの末尾に、必ず公式サイトへの誘導ボタン（CTA）をセンター配置で設置する。例外は認めない。

## Step 4: 投稿と最終点検
1. **網羅性チェック**: 表の数、本文の解説数、ボタン数がすべて一致しているか確認する。
2. **デプロイ**: `posts` テーブルへ反映し、Git プッシュ。
